Binary Search Tree (BST)

A Binary Search Tree is a hierarchical data structure where each node has at most two children. It maintains the binary search property: left children are smaller and right children are larger than their parent. This enables efficient searching, insertion, and deletion operations.

Key Operations:
  • Insertion: O(log n) average, O(n) worst-case
  • Search: O(log n) average, O(n) worst-case
  • Deletion: O(log n) average, O(n) worst-case
  • Traversal: In-order (LNR), Pre-order (NLR), Post-order (LRN)

Balanced BSTs (AVL, Red-Black) maintain O(log n) operations through automatic balancing.

Stack

A Stack is a linear LIFO (Last-In-First-Out) structure that supports two fundamental operations: push (add to top) and pop (remove from top). It's used in function calls, expression evaluation, undo mechanisms, and depth-first search algorithms.

Key Operations:
  • Push: O(1) - Adds element to stack top
  • Pop: O(1) - Removes and returns top element
  • Peek/Top: O(1) - Returns top element without removal
  • isEmpty: O(1) - Checks if stack is empty

All operations are O(1) because they only affect the top element.
